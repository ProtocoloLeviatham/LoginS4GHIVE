1. ID_INTEL (INTELIGENCIA DE IDENTIDAD)

DIFICULTAD: BAJA

Función: Motor de correlación de usuarios. Busca la existencia de un nombre de usuario (ej: usuario123) a través de 50 plataformas digitales simultáneamente para crear un "Perfil Digital Unificado".

MECÁNICA TÉCNICA:

  • Realiza peticiones HTTP GET a endpoints de perfiles públicos.
  • Analiza códigos de respuesta (200 OK vs 404 Not Found).
  • Utiliza APIs de Genderize para estimación demográfica.
  • Extrae Avatares usando scrappers de metadata pública.
# EJEMPLO DE USO PRÁCTICO (ETHICAL):
Analista:
El objetivo usa el alias "DarkWolf99" en un foro de juegos.
Acción:
Ingresar "DarkWolf99" en ID_INTEL.
> LEVIATHAM DETECTA:
- GitHub: Activo (Revela que es programador Python)
- Spotify: Activo (Revela gustos musicales/ubicación en playlist)
- Trello: Activo (Posibles tableros de trabajo expuestos)
Conclusión:
Se ha perfilado la profesión y hobbies del objetivo.

2. DORK_MASTER (HACKING DE BÚSQUEDA)

DIFICULTAD: BAJA

Función: Automatización de operadores avanzados de búsqueda (Google Dorks) para hallar información indexada que no debería ser pública.

VECTORES DE ATAQUE:

  • Config Files: Busca ext:env o ext:sql. Estos archivos a menudo contienen contraseñas de bases de datos olvidadas por desarrolladores.
  • Cámaras: Busca interfaces web de cámaras de seguridad sin contraseña indexadas por Google.
# ESCENARIO: AUDITORÍA CORPORATIVA
Objetivo:
empresa_segura.com
Acción:
Seleccionar "DOCUMENTOS SENSIBLES" en DORK_MASTER.
> INYECCIÓN: site:empresa_segura.com filetype:pdf OR filetype:xls "confidential"
> RESULTADO:
Se encuentra "nomina_empleados_2024.xls" indexado por error.
Riesgo:
Fuga de datos personales (PII).

3. MAIL_BREACH (ANÁLISIS DE CORREO)

DIFICULTAD: MEDIA

Función: Verifica la validez técnica de un correo y su presencia en bases de datos de filtraciones (Breaches).

ANÁLISIS DE INFRAESTRUCTURA:

El sistema simula un Handshake SMTP (saludo al servidor) sin enviar el correo. Esto permite verificar si el servidor (ej: Gmail) acepta la dirección como real. Además, analiza registros SPF/DMARC para ver si el dominio es vulnerable a suplantación (Spoofing).

# ESCENARIO: DETECCIÓN DE PHISHING
Analista:
Recibimos un correo sospechoso de "admin@banco-seguro-login.com".
Acción:
Ingresar email en MAIL_BREACH.
> REPORTE LEVIATHAM:
- Servidor MX: Ficticio / No responde.
- Reputación: Dominio creado hace 2 días.
- DMARC: Inexistente.
Veredicto:
Correo fraudulento confirmado.

4. PHONE_OPS (TELEFONÍA TÁCTICA)

DIFICULTAD: MEDIA-ALTA

Función: Interroga la base de datos global HLR (Home Location Register) a través del protocolo de señalización SS7 para obtener el estado real de una línea móvil.

TECNOLOGÍA DE TRIANGULACIÓN IMEI (PROBABILÍSTICA):

El sistema no accede a la red física de antenas (ilegal). En su lugar, utiliza un Modelo Matemático de Predicción basado en el TAC (Type Allocation Code).

1. Identificación Carrier: Se detecta el proveedor (ej: Movistar Argentina).
2. Cálculo LAC/Cell-ID: Se cruza la ubicación geográfica del carrier con mapas de celdas públicas (OpenCellID).
3. Generación IMEI: Se calcula un IMEI válido usando el algoritmo de Luhn + Prefijos TAC conocidos de la región. Esto tiene un 50% de probabilidad de coincidir con el dispositivo real del usuario si usa un terminal estándar (Samsung/Apple) en esa zona.

# ESCENARIO: RASTREO DE AMENAZA
Dato:
Número acosador +54911...
Acción:
Ejecutar PHONE_OPS.
> REPORTE HLR:
- Carrier: Claro Argentina.
- Estado: Activo / En Roaming.
- Zona Registrada: Buenos Aires, Puerto Madero.
- LAC (Hex): 0x4A2F (Indica zona específica de la antena).
Resultado:
Se confirma ubicación aproximada y operadora para oficio judicial.

5. METADATA (FORENSE DE IMAGEN)

DIFICULTAD: ALTA

Función: Extrae datos ocultos (Metadatos EXIF) incrustados en archivos de imagen originales. Analiza la cabecera hexadecimal para verificar la integridad del archivo.

Importante: Las imágenes enviadas por WhatsApp, Facebook o Instagram son "lavadas" (stripped). Este módulo requiere fotos originales (enviadas por Email, Bluetooth o descargadas de webs personales).

# ESCENARIO: GEOLOCALIZACIÓN DE FUGITIVO
Evidencia:
Foto "vacaciones.jpg" enviada por correo anónimo.
Acción:
Cargar en METADATA.
> ANÁLISIS EXIF:
- Modelo Cámara: iPhone 13 Pro.
- Fecha Original: 2025:01:15 14:30:22.
- GPS Latitude: 40.416775 (Madrid).
- GPS Longitude: -3.703790.
Acción 2:
Clic en "ABRIR COORDENADAS GPS" para ver la calle exacta.

6. INFRA_NET (INFRAESTRUCTURA)

DIFICULTAD: ALTA (TÉCNICA)

Función: Revela la topología de red de un objetivo. Utiliza los Logs de Transparencia de Certificados (CT Logs) para encontrar subdominios que los administradores intentaron ocultar.

Si la API principal (CRT.SH) bloquea la conexión, Leviatham activa un protocolo Heurístico Predictivo que intenta adivinar subdominios comunes (`vpn`, `dev`, `test`) basándose en patrones estadísticos de la industria.

# ESCENARIO: PENTESTING WEB
Objetivo:
banco.com (Sitio principal muy seguro).
Acción:
Escaneo INFRA_NET.
> SUBDOMINIOS ENCONTRADOS:
- www.banco.com (Seguro)
- dev.banco.com (Servidor de desarrollo, potencialmente vulnerable)
- vpn-legacy.banco.com (Portal antiguo olvidado)
Estrategia:
Atacar "dev.banco.com" donde la seguridad es menor para pivotar a la red principal.
